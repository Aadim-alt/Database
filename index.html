üîí Constraints & Data Integrity
‚ñº

PRIMARY KEY

-- During creation CREATE TABLE Theatre ( Movie_ID INT PRIMARY KEY AUTO_INCREMENT, Movie_Name VARCHAR(100) ); -- After creation ALTER TABLE Members ADD CONSTRAINT PRIMARY KEY(Member_ID);

FOREIGN KEY

ALTER TABLE transactions ADD CONSTRAINT fk_member FOREIGN KEY(Member_ID) REFERENCES Members(Member_ID);
Remember: Use REFERENCES (with 'S'), not 'reference'

Other Constraints

-- UNIQUE ALTER TABLE Members ADD CONSTRAINT UNIQUE(Member_ID); -- NOT NULL ALTER TABLE Members MODIFY Name VARCHAR(100) NOT NULL; -- CHECK ALTER TABLE Members ADD CONSTRAINT chk_donation CHECK(Monthly_Donation > 50); -- DEFAULT ALTER TABLE Members ALTER Total_Donation SET DEFAULT 0;

AUTO_INCREMENT

-- Must be PRIMARY KEY or UNIQUE first ALTER TABLE Customers ADD CONSTRAINT PRIMARY KEY(Customer_ID); -- Then add auto_increment ALTER TABLE Customers MODIFY Customer_ID INT AUTO_INCREMENT; -- Set starting value ALTER TABLE Customers AUTO_INCREMENT = 100; -- Reset to 1 ALTER TABLE Customers AUTO_INCREMENT = 1;
üí° Benefit: No need to specify ID when inserting - it auto-generates!
üîó Table Joins
‚ñº

INNER JOIN

SELECT * FROM transactions INNER JOIN Members ON transactions.Member_ID = Members.Member_ID;
üìå Returns: Only matching records from BOTH tables

LEFT JOIN

SELECT * FROM Members LEFT JOIN transactions ON Members.Member_ID = transactions.Member_ID;
üìå Returns: ALL from left table + matching from right (NULL if no match)

RIGHT JOIN

SELECT * FROM transactions RIGHT JOIN Members ON transactions.Member_ID = Members.Member_ID;
üìå Returns: ALL from right table + matching from left (NULL if no match)

Quick Comparison

Join Type When to Use
INNER JOIN Only want records that exist in both tables
LEFT JOIN Want all from first table, even if no match
RIGHT JOIN Want all from second table, even if no match

üéØ MariaDB Practice Quiz

Test your knowledge with these important questions!

Question 1
What is the correct syntax to use a database in MariaDB?
‚úÖ Correct Answer: B
The correct syntax is USE database_name; without the word "database" before the name.
Question 2
How do you check for NULL values in a WHERE clause?
‚úÖ Correct Answer: C
You must use IS NULL to check for NULL values. The equality operator (=) doesn't work with NULL.
Question 3
Which keyword is used in FOREIGN KEY constraint?
‚úÖ Correct Answer: B
The correct keyword is REFERENCES (with an 'S' at the end), not "reference".
Question 4
What does INNER JOIN return?
‚úÖ Correct Answer: B
INNER JOIN returns only the records that have matching values in both tables.
Question 5
Before using AUTO_INCREMENT, what must the column be?
‚úÖ Correct Answer: B
AUTO_INCREMENT requires the column to be either a PRIMARY KEY or have a UNIQUE constraint.
Question 6
What happens if you use DELETE FROM table_name without a WHERE clause?
‚úÖ Correct Answer: C
DELETE without WHERE clause removes ALL rows from the table. Always use WHERE clause to be safe!
Question 7
Which ALTER TABLE command is used to rename a column?
‚úÖ Correct Answer: B
Use ALTER TABLE table_name CHANGE old_name new_name datatype; to rename a column.
Question 8
Which function returns the current date and time together?
‚úÖ Correct Answer: C
NOW() returns the current date and time in DATETIME format (YYYY-MM-DD HH:MM:SS).
Question 9
What is the correct syntax to update a specific row?
‚úÖ Correct Answer: B
Correct syntax: UPDATE table_name SET column = value WHERE condition; Don't use "table" before the table name.
Question 10
Are table names case-sensitive in MariaDB?
‚úÖ Correct Answer: B
Table names ARE case-sensitive in MariaDB. "Members" and "members" are different tables!
