1. Database Operations

Creating a Database

CREATE DATABASE Build_And_Ship;

Using a Database

USE Build_And_Ship;
‚ö†Ô∏è Important: Use USE database_name (not USE database database_name)

2. Table Creation & Structure

Basic Table Creation

CREATE TABLE Members ( Name VARCHAR(100), Post VARCHAR(200), Member_ID INT );
üìù Key Points:
  • Table must have at least one column
  • Each column needs a data type
  • No trailing comma after last column

View Table Structure

DESC Members;

View Table Data

SELECT * FROM Members;
‚ö†Ô∏è Important: Table names are case-sensitive in MariaDB

Modifying Table Structure (ALTER TABLE)

Reordering Columns

ALTER TABLE Members MODIFY COLUMN Member_ID INT FIRST;

Renaming Columns

ALTER TABLE Members CHANGE Post Position VARCHAR(200);

Adding Columns

-- Add column at the end ALTER TABLE Members ADD COLUMN Donation INT; -- Add column after specific column ALTER TABLE Members ADD COLUMN Monthly_Donation INT AFTER Position;

3. Data Operations

Inserting Data

Insert Multiple Rows

INSERT INTO Members VALUES (1, "Aadim Dhakal", "Main Lead"), (2, "Samriddha Pant", "Vice Lead"), (3, "Chandan Shah", "Tech Lead"), (4, "Abhas Shakya", "Member");

Insert Specific Columns

INSERT INTO Customers(Customer_Name) VALUES ("Aadim"), ("Palpasa"), ("Shine");

Updating Data

Update Single Row

UPDATE Members SET Donation = 500 WHERE Member_ID = 1;

Update All Rows

UPDATE Members SET Monthly_Donation = 500;
‚ö†Ô∏è Important: Don't use UPDATE table table_name - just UPDATE table_name

Deleting Data

Delete Single Row

DELETE FROM Members WHERE Member_ID = 4;

Delete All Rows

DELETE FROM Members;
‚ö†Ô∏è Warning: Always use WHERE clause when deleting to avoid removing all data!

Working with NULL Values

‚ùå Incorrect Way:
SELECT * FROM Members WHERE Total_Donation = NULL; -- Returns empty set
‚úÖ Correct Way:
SELECT * FROM Members WHERE Total_Donation IS NULL;

üìù Result Example:

5. Constraints

UNIQUE Constraint

ALTER TABLE Members ADD CONSTRAINT UNIQUE(Member_ID);

PRIMARY KEY Constraint

-- Method 1: During table creation CREATE TABLE Theatre ( Movie_ID INT PRIMARY KEY AUTO_INCREMENT, Movie_Name VARCHAR(100) ); -- Method 2: After table creation ALTER TABLE Members ADD CONSTRAINT PRIMARY KEY(Member_ID);

NOT NULL Constraint

ALTER TABLE Members MODIFY Name VARCHAR(100) NOT NULL;

CHECK Constraint

ALTER TABLE Members ADD CONSTRAINT chk_donation CHECK(Monthly_Donation > 50);

DEFAULT Constraint

ALTER TABLE Members ALTER Total_Donation SET DEFAULT 0;

FOREIGN KEY Constraint

ALTER TABLE transactions ADD CONSTRAINT transaction_fk FOREIGN KEY(Member_ID) REFERENCES Members(Member_ID);
‚ö†Ô∏è Important Notes:
  • AUTO_INCREMENT requires the column to be a PRIMARY KEY or UNIQUE
  • Foreign key must reference a PRIMARY KEY or UNIQUE column in parent table

AUTO_INCREMENT

-- First make it a primary key ALTER TABLE Customers ADD CONSTRAINT PRIMARY KEY(Customer_ID); -- Then add auto_increment ALTER TABLE Customers MODIFY Customer_ID INT AUTO_INCREMENT;
‚úÖ Benefit: No need to specify ID when inserting - it auto-generates

Setting AUTO_INCREMENT Value

Set Starting Value

-- Set AUTO_INCREMENT to start from 100 ALTER TABLE Customers AUTO_INCREMENT = 100;

Reset AUTO_INCREMENT

-- Reset to start from 1 ALTER TABLE Customers AUTO_INCREMENT = 1;
üìù When to use:
  • After deleting rows to reset the counter
  • To set a specific starting number for IDs
  • To maintain sequential ID numbering

6. Table Joins

INNER JOIN

SELECT * FROM transactions INNER JOIN Members ON transactions.Member_ID = Members.Member_ID;
üìù What it does: Returns only matching records from both tables

LEFT JOIN (LEFT OUTER JOIN)

SELECT * FROM Members LEFT JOIN transactions ON Members.Member_ID = transactions.Member_ID;
üìù What it does: Returns all records from the left table (Members), and matching records from the right table. If no match, NULL values are shown for right table columns.

RIGHT JOIN (RIGHT OUTER JOIN)

SELECT * FROM transactions RIGHT JOIN Members ON transactions.Member_ID = Members.Member_ID;
üìù What it does: Returns all records from the right table (Members), and matching records from the left table. If no match, NULL values are shown for left table columns.

Selecting Specific Columns from Join

SELECT Transaction_ID, Transaction_Amount, Name, Email FROM transactions INNER JOIN Members ON transactions.Member_ID = Members.Member_ID;
‚ö†Ô∏è Important: Use correct table names in ON clause (e.g., Customers.Customer_ID, not Customer.Customer_ID)

JOIN Types Comparison

Join Type Description Use Case
INNER JOIN Returns only matching records from both tables When you want only records that exist in both tables
LEFT JOIN Returns all from left table + matching from right When you want all records from first table regardless of match
RIGHT JOIN Returns all from right table + matching from left When you want all records from second table regardless of match

7. Transaction Control

Disable Auto-commit

SET AUTOCOMMIT = OFF;

Commit Changes

COMMIT;

üéØ Best Practices:
  • Use consistent naming conventions (PascalCase or snake_case)
  • Always specify column data types when creating tables
  • Use WHERE clause carefully when updating to avoid unintended changes

  • Use FOREIGN KEY to maintain referential integrity
  • Add constraints to ensure data quality
  • Test queries before executing on production data
  • Use transactions for critical operations
